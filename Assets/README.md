## Pesquisa
  CRUD é um conceito bem simples e muito usado em sistemas feitos com PHP. Ele representa as quatro ações básicas que você pode fazer com os dados: criar, ler, atualizar e excluir. Essa ações seriam usadas da seguinte forma no nosso projeto:
 
## Create:
  É quando você adiciona um novo dado ao sistema. Na tela de cadastro, quando um usuário preenche nome, e-mail e senha e clica em “Cadastrar”, o sistema usa o Create para salvar essas informações no banco de dados.
 
## Read:
  Essa ação serve para visualizar ou consultar os dados que já estão armazenados. Na tela de login, o sistema precisa verificar se o usuário existe.
 
## Update:
  É usada quando você precisa modificar alguma informação existente. Se o usuário esquecer a senha e criar uma nova, o sistema atualiza esse dado no banco.  
 
## Delete :
  Essa ação remove um dado do sistema. Caso exista uma função de remover conta ou um administrador possa excluir usuários
 
## Resumo:
  Em PHP, o CRUD funciona como uma ponte entre o sistema e o banco de dados, permitindo cadastrar, visualizar, editar e excluir informações de forma dinâmica.

# Pesquisa sobre SCRUM

## Introdução
  O SCRUM é uma metodologia ágil muito utilizada no desenvolvimento de software. Seu objetivo é organizar o trabalho em equipe, melhorar a comunicação e aumentar a produtividade durante o desenvolvimento de projetos.

## O que é SCRUM
  O Scrum é uma das estruturas ágeis mais populares, pois ajuda as equipes a lidarem com projetos complexos dividindo o trabalho em ciclos menores e interativos chamados *Sprints*. Ele foi criado para melhorar a colaboração entre os integrantes da equipe, aumentar a transparência e promover a melhoria contínua no desenvolvimento de projetos.

## Objetivo da metodologia
  O objetivo do SCRUM é facilitar o trabalho da equipe, melhorar a comunicação e entregar resultados com mais qualidade em menos tempo.

## Principais papéis

### Product Owner (PO)
  É a pessoa responsável por definir o que precisa ser feito no projeto. Também representa o cliente e organiza as prioridades das tarefas.

### Scrum Master
  É o responsável por ajudar a equipe a seguir corretamente a metodologia SCRUM, além de resolver problemas que possam atrapalhar o desenvolvimento do projeto.

### Development Team (Equipe de Desenvolvimento)
  É a equipe responsável por desenvolver as funcionalidades do sistema. Pode ser formada por programadores, designers, testadores e outros profissionais envolvidos no projeto.

## Eventos do SCRUM

### Sprint
  É um período de trabalho, geralmente entre 1 e 4 semanas, onde a equipe desenvolve parte do projeto.

### Sprint Planning
  Reunião realizada no início da Sprint para planejar as tarefas que serão desenvolvidas.

### Daily Scrum
  Reunião rápida diária onde a equipe conversa sobre o andamento do projeto, atividades realizadas e dificuldades encontradas.

### Sprint Review
  Reunião realizada no final da Sprint para apresentar o que foi desenvolvido durante o período.

### Sprint Retrospective
  Reunião onde a equipe analisa o que deu certo e o que pode ser melhorado nos próximos ciclos.

## Benefícios do uso do SCRUM em projetos de software
- Melhor organização do trabalho;
- Melhor comunicação entre os integrantes da equipe;
- Entregas mais rápidas;
- Aumento da produtividade;
- Facilidade para realizar mudanças no projeto;
- Melhor qualidade no desenvolvimento do sistema.

## Conclusão
  Concluímos que o SCRUM é uma metodologia muito importante para o desenvolvimento de projetos de software, pois ajuda na organização das tarefas, melhora o trabalho em equipe e permite entregas mais rápidas e eficientes.

##PESQUISA  SOBRE O XAMPP                                

## 1.O que é Xampp?

R: O XAMPP é um pacote com os principais servidores de código aberto do mercado, incluindo FTP, banco de dados MySQL e Apache com suporte às linguagens PHP e Perl. Com ele, é possível rodar sistemas como WordPress e Drupal localmente, o que facilita e agiliza o desenvolvimento. 

## 2. Principais componentes do ambiente: 

R: Apache, MySQL/MariaDB, PHP, phpMyAdmin.

## 3. Finalidade de cada componente: 

R:  Apache: servidor web responsável por abrir e exibir os projetos no navegador através do localhost.

MySQL/MariaDB: banco de dados utilizado para armazenar e organizar informações das aplicações.

PHP: linguagem de programação usada para criar páginas dinâmicas e sistemas web.

phpMyAdmin: ferramenta visual que permite administrar bancos de dados de forma simples e prática.

## 4. Como realizar a instalação e configuração básica: 

R: Acesse o Site Oficial do XAMPP  ,baixe o instalador no site oficial e Selecione os componentes que deseja instalar. É recomendado deixar todos marcados (Apache, MySQL, PHP, phpMyAdmin, etc.) e execute, mantenha o diretório padrão e avance. Abra o painel de controle e clique em "Start" ao lado do Apache e do MySCL para iniciar o seu servidor local. 
      Você pode gerenciar seu banco de dados clicando no botão Admin ao lado do MySQL no painel de controle ou digitando http://localhost/phpmyadmin diretamente no seu navegador. 

## 5. Importância do ambiente para desenvolvimento local:

R: O XAMPP é importante porque permite criar, testar e executar sites e sistemas diretamente no computador, sem precisar de hospedagem na internet. Ele facilita o desenvolvimento de aplicações PHP ao reunir ferramentas como Apache, MariaDB e PHP em um único ambiente, tornando os testes mais rápidos, seguros e práticos para os desenvolvedores.
